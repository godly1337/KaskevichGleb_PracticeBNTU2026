Как выполнялась работа


ЗАДАНИЕ

Анализ инерциальных данных (IMU) мобильного робота: загрузить данные
акселерометра и гироскопа, построить графики, определить моменты
старта, остановки и резких поворотов, сделать вывод о характере
движения робота.


1. ПОИСК ДАННЫХ

Для работы использовался датасет MAGF-ID (Multiple and Gyro-Free
Inertial Datasets), ссылки на который были указаны в задании:

репозиторий: https://github.com/ansfl/MAGF-ID
статья: https://link.springer.com/article/10.1038/s41597-024-03917-6

Из репозитория был скачан CSV-файл с показаниями акселерометра и
гироскопа мобильного робота.


2. НАПИСАНИЕ СКРИПТА ОБРАБОТКИ ДАННЫХ

Был написан скрипт на Python (code.py), который:

- читает CSV-файл и автоматически определяет, какие столбцы относятся
  к ускорениям (ax, ay, az) и угловым скоростям (gx, gy, gz);
- строит графики ускорений по трём осям и графики угловых скоростей
  по трём осям;
- вычисляет модуль ускорения и модуль угловой скорости — по ним проще
  находить моменты резких изменений в движении;
- ищет резкие изменения: скачок продольного ускорения считается
  стартом (если ускорение растёт) или остановкой (если падает), а
  всплеск угловой скорости — поворотом;
- отмечает найденные события прямо на графиках подписанными
  вертикальными линиями (старт — зелёным, остановка — красным,
  поворот — оранжевым);
- выводит в консоль время каждого найденного события.

Использованные библиотеки: pandas (чтение CSV и работа с таблицей
данных), numpy (вычисления), matplotlib (построение графиков), scipy
(поиск пиков на графике для определения событий).


3. НАСТРОЙКА ОКРУЖЕНИЯ И ЗАПУСК

Работа велась в Visual Studio Code. Для запуска скрипта потребовалось:

- установить нужные библиотеки командой:
  pip install pandas numpy matplotlib scipy


4. РЕЗУЛЬТАТ РАБОТЫ СКРИПТА

После успешного запуска скрипт создал папку output_plots с тремя
графиками:

- accelerometer.png — ускорения по осям X, Y, Z и общий модуль
  ускорения;
- gyroscope.png — угловые скорости по осям X, Y, Z и общий модуль
  угловой скорости;
- overview.png — сводный график, на котором модуль ускорения и модуль
  угловой скорости показаны вместе, с отмеченными моментами старта,
  остановки и поворотов.

В консоли скрипт вывел точное время каждого обнаруженного события.


5. ВЫВОД ПО ХАРАКТЕРУ ДВИЖЕНИЯ

В состоянии покоя показания акселерометра держатся на уровне
ускорения свободного падения по вертикальной оси, а показания
гироскопа колеблются около нуля. В моменты старта и остановки
наблюдаются короткие резкие скачки продольного ускорения:
положительный при разгоне и отрицательный при торможении, при этом
показания гироскопа в эти моменты почти не меняются. Во время
поворотов, наоборот, резко возрастает угловая скорость (в первую
очередь по вертикальной оси), а ускорение при этом меняется не так
сильно. Таким образом, по совместному анализу данных акселерометра и
гироскопа можно чётко различать фазы прямолинейного движения,
разгона, торможения и поворотов робота.


6. ЗАГРУЗКА ПРОЕКТА НА GITHUB

Готовый код и графики были выложены в отдельный репозиторий на
GitHub. Для этого выполнялись следующие шаги:

1. Создан новый репозиторий на github.
2. В папке проекта в терминале выполнена инициализация git:
   git init
3. Все файлы проекта добавлены в индекс:
   git add .
4. Сделан первый коммит:
   git commit -m "Первая версия: анализ IMU данных робота"
5. Локальная папка привязана к репозиторию на GitHub:
   git remote add origin <ссылка на репозиторий>
   git branch -M main
6. Файлы отправлены на GitHub:
   git push -u origin main
